- 既に作成済みなので、下記を参考にcit-coin-uiリポジトリのkeysディレクトリにサービスアカウントキーをダウンロードしてください
    - GCP
        - スプレッドシート操作用のサービスアカウント
            - [https://console.cloud.google.com/storage/browser/_details/pitpa-staging-keys/pitpa-staging-cit-coin-demo.json;tab=live_object?hl=ja&inv=1&invt=AbmHfg&project=pitpa-staging](https://console.cloud.google.com/storage/browser/_details/pitpa-staging-keys/pitpa-staging-cit-coin-demo.json;tab=live_object?hl=ja&inv=1&invt=AbmHfg&project=pitpa-staging)
        - 許可するAPI
            - Google Sheets API
    - サービスアカウントのjsonファイルをダウンロードしてエンコードする
        - 下記の一連のコマンドを実行する
            
            ```bash
            gcloud auth login
            gcloud config set account hokuto-morita@dou.id
            gcloud config set project pitpa-staging
            
            pwd
            => ~/<workspace>/cit-coin-ui
            
            gcloud storage cp gs://pitpa-staging-keys/pitpa-staging-cit-coin-demo.json keys/google-sheets-service-account.json
            
            yarn encode-keys --file keys/google-sheets-service-account.json
            ```
            
            - `hokuto-morita@dou.id`の部分はpitpa-stagingプロジェクトにアクセスできる各自のアカウントのメールアドレスを指定してください

## 1. cit-coin-contractの各種スマートコントラクトのデプロイ

各種スマートコントラクトをデプロイする前のcit-coin-contractの`.env`の状態

まずは、下記操作を実行して.envの雛形を作ってください！

```bash
pwd
=> ~/<workspace>/cit-coin-contract

cp .env.example .env
```

次は、`.env`の中身を下記の状態にしてください！

```bash
#===============================================[ DEPLOYMENT SETTINGS ]===============================================
## Gnosis Safeは使用しないことにしたので、開発用の共有ウォレットアドレスを指定する
GNOSIS_OWNER="0x75E0B6F7AbE738E92f9c4eBD836DFde787f22eC5"
## この環境変数は使用してない可能性があるのでこのままで良いかも
GATEKEEPER="0x0000000000000000000000000000000000000000"
## 開発用の共有ウォレットアドレスの秘密鍵を設定する
PRIVATE_KEY="<notionページ「cit-coinの開発に使用する機密情報」に記載されてる開発用の共有ウォレットアドレスの秘密鍵を貼り付けてください>"
# ================================================[ OPTIMISM SETTINGS ]================================================
# grab this API KEY from https://optimistic.etherscan.io/myapikey
OPTIMISM_ETHERSCAN_API_KEY="<notionページ「cit-coinの開発に使用する機密情報」に記載されてるOptimismのetherscanのAPIキーを貼り付けてください>"
# =============================================[ LEARN TO EARN SETTINGS ]=============================================
## デプロイしたCJPYコントラクトアドレス
CJPY_ADDRESS=""
## FUND_ADDRESSはFaucetコントラクト経由でPOLを送金する方のアドレス
### 基本的には環境変数GNOSIS_OWNERと同じアドレスを設定する
FUND_ADDRESS="0x75E0B6F7AbE738E92f9c4eBD836DFde787f22eC5"
## デプロイしたRegistryコントラクトアドレス
REGISTRY_ADDRESS=""
## デプロイしたLearnToWarnコントラクトアドレス
LEARN_TO_WARN_ADDRESS=""

```

- 関連するnotionページ
    - [cit-coinで使用する秘密鍵などの機密情報](https://www.notion.so/cit-coin-1766aec3049580c0ba04e46ff157f786?pvs=21)
- 補足説明
    - 下記の環境変数は各スマートコントラクトをデプロイ後に追記してください
        - REGISTRY_ADDRESS
        - CJPY_ADDRESS
        - LEARN_TO_WARN_ADDRESS

## 3. (開発用の共有ウォレットでcJPYが不足してる場合)cJPYをmintする

下記シーケンスのように学生さんがクイズに正答すると、報酬としてcJPYが学生さんに払われます

- [Answering Questions by Student (学生が質問に答える)](https://www.notion.so/Answering-Questions-by-Student-a145745aa51846ec872fd49f89f6b76a?pvs=21)

そして、そのcJPYは誰が持ってて誰が払うのか？

⇒ LearnToEarn コントラクトデプロイ時にデプロイスクリプト内で指定されてる`環境変数GNOSIS_OWNER`に指定されてるウォレットから支払われます！

つまり、基本的には開発用の共有ウォレットからcJPYを学生に支払います。

そのため、開発用の共有ウォレットはcJPYを大量に保有してる必要があります。

開発用の共有ウォレットのcJPYの保有状況の確認方法

- sepolia-optimism.etherscanに開発用の共有ウォレットアドレスを入力することで確認できます
    - [https://sepolia-optimism.etherscan.io/address/0x75E0B6F7AbE738E92f9c4eBD836DFde787f22eC5](https://sepolia-optimism.etherscan.io/address/0x75E0B6F7AbE738E92f9c4eBD836DFde787f22eC5)

## 1. cit-coin-contractの各種スマートコントラクトのデプロイ

各種スマートコントラクトをデプロイする前のcit-coin-contractの`.env`の状態

まずは、下記操作を実行して.envの雛形を作ってください！

```bash
pwd
=> ~/<workspace>/cit-coin-contract

cp .env.example .env
```

次は、`.env`の中身を下記の状態にしてください！

```bash
#===============================================[ DEPLOYMENT SETTINGS ]===============================================
## Gnosis Safeは使用しないことにしたので、開発用の共有ウォレットアドレスを指定する
GNOSIS_OWNER="0x75E0B6F7AbE738E92f9c4eBD836DFde787f22eC5"
## この環境変数は使用してない可能性があるのでこのままで良いかも
GATEKEEPER="0x0000000000000000000000000000000000000000"
## 開発用の共有ウォレットアドレスの秘密鍵を設定する
PRIVATE_KEY="<notionページ「cit-coinの開発に使用する機密情報」に記載されてる開発用の共有ウォレットアドレスの秘密鍵を貼り付けてください>"
# ================================================[ POLYGON SETTINGS ]================================================
## PolygonscanのAPIキーもops privateで管理した方がいいかもですが
POLYGONSCAN_API_KEY="<notionページ「cit-coinの開発に使用する機密情報」に記載されてるPolygonscanのAPIキーを貼り付けてください>"
# =============================================[ LEARN TO EARN SETTINGS ]=============================================
## デプロイしたCJPYコントラクトアドレス
CJPY_ADDRESS=""
## FUND_ADDRESSはFaucetコントラクト経由でPOLを送金する方のアドレス
### 基本的には環境変数GNOSIS_OWNERと同じアドレスを設定する
FUND_ADDRESS="0x75E0B6F7AbE738E92f9c4eBD836DFde787f22eC5"
## デプロイしたRegistryコントラクトアドレス
REGISTRY_ADDRESS=""
## デプロイしたLearnToWarnコントラクトアドレス
LEARN_TO_WARN_ADDRESS=""

```

- 関連するnotionページ
    - [cit-coinで使用する秘密鍵などの機密情報](https://www.notion.so/cit-coin-1766aec3049580c0ba04e46ff157f786?pvs=21)
- 補足説明
    - 下記の環境変数は各スマートコントラクトをデプロイ後に追記してください
        - REGISTRY_ADDRESS

        - LEARN_TO_WARN_ADDRESS

## 3. (開発用の共有ウォレットでcJPYが不足してる場合)cJPYをmintする

下記シーケンスのように学生さんがクイズに正答すると、報酬としてcJPYが学生さんに払われます

- [Answering Questions by Student (学生が質問に答える)](https://www.notion.so/Answering-Questions-by-Student-a145745aa51846ec872fd49f89f6b76a?pvs=21)

そして、そのcJPYは誰が持ってて誰が払うのか？

⇒ LearnToEarn コントラクトデプロイ時にデプロイスクリプト内で指定されてる`環境変数GNOSIS_OWNER`に指定されてるウォレットから支払われます！

つまり、基本的には開発用の共有ウォレットからcJPYを学生に支払います。

そのため、開発用の共有ウォレットはcJPYを大量に保有してる必要があります。

開発用の共有ウォレットのcJPYの保有状況の確認方法

- polygonscanに開発用の共有ウォレットアドレスを入力することで確認できます
    - [https://amoy.polygonscan.com/address/0x75E0B6F7AbE738E92f9c4eBD836DFde787f22eC5](https://amoy.polygonscan.com/address/0x75E0B6F7AbE738E92f9c4eBD836DFde787f22eC5)

    - cJPYの送金履歴

# cit-coin-uiの操作時のTips

- cit-coin-uiでは学生追加時やAdmin権限追加時などにブロックチェーン(スマートコントラクト)のトランザクションが実行されます(他の操作でもトランザクションが実行されます)
    - その際に、**たまに失敗する**ことがあります
        - その場合は、何度かやり直してください。おそらくいつか成功します
        - 何度やっても成功しない場合は、エラーログなどを仕込んで原因を探ってください
